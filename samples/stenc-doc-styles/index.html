Zero dependency style study

Three documentation page structures for Stenc

These samples avoid framework-specific patterns and use plain HTML and CSS. The goal is to choose a fixed Stenc document style that is easy to install, easy to scan, and still precise enough for agents. The section model follows Superpowers-style specs and plans.

Recommendation

Use Evidence-led Contract as the default Stenc document page. It keeps the Superpowers-style sequence of goal, architecture, scope, tasks, validation, and agent handoff visible as first-class content.

Reference principles used

  • Small-screen-first layouts and constrained line length.
  • Consistent type scale and vertical rhythm.
  • Grid structure for repeatable dense information layouts.
  • Accessibility and readable typography over visual novelty.

Component patterns added to the samples

Human reading aids

Callouts, scope boxes, key terms, examples, and review checklists.

Agent execution aids

Plan headers, task checkboxes, source paths, validation commands, and handoff blocks.

Shared evidence

Decision records, surface maps, risk tables, and open question lists.
