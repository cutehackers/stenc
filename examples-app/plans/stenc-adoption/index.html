Rail Stenc Adoption Plan

Example single-document Stenc plan for adopting Stenc in Rail docs.

Status: draft Owner: rail-docs Updated: 2026-05-19

Human Summary

Adopt Stenc by converting a small set of canonical Rail docs first, then expand only after the template works for real runtime contracts.

Agent Summary

Start from canonical specs and plans. Preserve Rail's source-of-truth hierarchy and add Stenc validation before broad migration.

Goal

Make Rail specs and plans easier for humans to scan while keeping them structured enough for AI coding agents to follow.

Architecture

Convert one canonical spec and one implementation plan into structured JSON, then render both through the fixed Stenc static app.
