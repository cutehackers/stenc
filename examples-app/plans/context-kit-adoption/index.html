Rail ContextKit Adoption Plan

Example single-document ContextKit plan for adopting ContextKit in Rail docs.

Status: draft Owner: rail-docs Updated: 2026-05-19

Human Summary

Adopt ContextKit by converting a small set of canonical Rail docs first, then expand only after the template works for real runtime contracts.

Agent Summary

Start from canonical specs and plans. Preserve Rail's source-of-truth hierarchy and add ContextKit validation before broad migration.

Source Of Truth

  • docs/SPEC.md
  • docs/ARCHITECTURE.md
  • docs/CONVENTIONS.md

Related Spec

spec:artifact-identity

Goal

Make Rail specs and plans easier for humans to scan while keeping them structured enough for AI coding agents to follow.

Scope

In

  • One canonical spec example
  • One implementation plan example
  • Thin AGENTS.md linkage if adopted in Rail

Out

  • Full docs migration
  • Runtime behavior changes

Current State

Rail docs are plain Markdown with strong contracts but limited visual scanning support.

Target State

Rail docs use one JSON file per spec or plan and a fixed web page for human reading.
